Structure du Code

Coté Client:

1. connextion.html

Permet au client de ce connecter (session) après une batterie de tests au niveau du serveur :

  • Test des champs email et password (non vide).
  • Regex de vérification du format de l'email.
  • Verification de L'existance de l'Email dans la base de donnée.
  • Test d'égalité du mot de passe entré et de celui de la base de donnée (après Hash).

2. inscription.html

Permet au client de s'inscrire au site, après inscription il sera renvoyer vers la page connexion, voici les test de sécurité:

  • On utilise htmlspecialchars sur tout les champs pour éviter les injections.
  • Test de la taille de tout les champs (pseudo et password doit etres compris en 6 et 25).
  • Regex de vérification du format du pseudo et de l'email.
  • Test de l'existance du pseudo et de l'email dans la base de donnée.
  • Test d'égalité entre les deux mots de passe entrés.

Structure du Code

Coté Client:

3. liste.html

Permet d'affichier la liste des Utilisateurs connectés et de les défier, il permet aussi d'accepter les défis des autres membres (utilisations des websocket pour dialoguer).

4. profil.html

On peut y consulter notre profil : pseudo, email, nombres de défaites et de victoires.

5. changePassword.html

On peut changer son mot de passe utilisateur autant de fois qu'on le veut, à conditions d'inserer un nouveau mot de passe, (on reprend les meme test que dans inscription)

6. jeuClient.html

Permet de jouer contre un adversaire, Cette page gère uniquement l'affichage du jeu et les cliques des joueurs.

Structure du Code

Coté Serveur:

1. User.js

La classe User représente un utilisateur connecter, une instance d'user est créer a chaque nouvelle connexion.

2. dames.js

La classe Dames représente le jeu de Dame: elle implemente les régles international du jeu de Dames.

3. db_init.js

Ici on initialise la base de donnée MongoDB.

4. serveur.js

Le serveur.js gére toute les requêtes get et post ainsi que les messages websocket et utilisateurs.

5. package.json

Ce fichier contient les différentes configuration du projet.

Technologies utilisées

Front-End

  • HTML5
  • CSS3 : responsive design, flexbox, media queries.
  • JAVASCRIPT : Affichage coté client.
  • NUNJUCKS : affiche les données envoyer par le serveur.

Back-End

  • Node.JS : Serveur.
  • EXPRESS : Framework Node.js.
  • SESSION : Identifier un utilisateur.
  • WEBSOCKET : Dialogue client serveur.

Base-De-Donnée

MongoDB

Méthodologie de Travail

  • Travail en coordination sur le même projet glitch.

  • Une personne travail sur le front-end et une sur le back-end pour finir rapidement les parties concernées.

  • Une sauvegarde du projet est faite regulierement sur Github.

  • Une communication parfaite au sein du groupe pour que chacun de nous sois capable de faire fonctionner correctement des modules non achevés.
